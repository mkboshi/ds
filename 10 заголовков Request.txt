Response headers:

1.Allow: Какие методы запроса поддерживает сервер (например, GET, POST и т. Д.)
2.Date: Текущее время по Гринвичу. Вы можете использовать setDateHeader для установки этого заголовка, чтобы избежать проблем с преобразованием формата времени.
3.X-xss-protection : может предотвратить некоторые XSS-атаки.
4.Server: Имя сервера. Сервлет обычно не устанавливает это значение, но устанавливается самим веб-сервером.
5.ETag: это токен, который можно связать с веб-ресурсами.
6.Set-Cookie: Установите файлы cookie, связанные со страницей.
7.Location: Указывает, куда клиент должен пойти, чтобы извлечь документ.
8.Last-Modified: Время последнего изменения документа.
9.Content-Encoding: Метод кодирования (Encode)документа. Только после декодирования можно получить тип контента, указанный в заголовке Content-Type.
10.Content-Type:Указывает, к какому типу MIME принадлежит следующий документ.

Request headers：

1.Accept: Сообщите веб-серверу, какой тип носителя он принимает, * / * означает любой тип, тип / * означает все подтипы этого типа, тип / подтип.
2.Accept-Charset: Браузер сообщает серверу, какой набор символов он может получить.
3.If-Unmodified-Since: Если запрошенный объект не был изменен по истечении времени, указанного в заголовке, запрошенное действие (например, возврат объекта) будет выполнено.
4.If-None-Match: Если ETag объекта изменяется, это фактически означает, что объект также изменился до того, как запрошенное действие будет выполнено и документ будет получен.
5.Range: Браузер (например, многопоточная загрузка Flashget) сообщает WEB-серверу, какую часть объекта он хочет получить. Например: Диапазон: байты = 1173546
6.Host: Клиент указывает доменное имя / IP-адрес и номер порта веб-сервера, к которому он хочет получить доступ. Например, Host: rss.sina.com.cn
7.Accept-Language:Браузер объявляет получаемый язык
8.Sec-ch-ua-platform : предоставляет серверу информацию о версии платформы, на которой выполняется данный пользовательский агент.
9.Network Error Logging : используется для настройки ведения журнала сетевых запросов
10.Proxy-Authenticate: Прокси-сервер отвечает браузеру и просит его предоставить информацию для проверки подлинности прокси.
